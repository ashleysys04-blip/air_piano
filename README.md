# AIR — 손 제스처 피아노 작곡기

손만으로 피아노를 연주하는 브라우저 앱입니다. 웹캠으로 양손을 인식해 오른손으로 멜로디를 연주하고, 왼손으로 셈여림·페달·화음을 제어합니다. 5명의 작곡가 어법 중 하나를 골라 그 스타일로 반주가 자동 생성됩니다.

---

## 실행 방법

카메라는 보안 정책상 `https://` 또는 `localhost`에서만 작동합니다.

```bash
# 프로젝트 폴더에서 실행
python3 -m http.server
```

크롬(Chrome)에서 아래 주소 열기:

```
http://localhost:8000/air-classical.html
```

1. **연주 시작** 버튼 클릭 → 피아노 음원 다운로드 + 카메라 권한 요청
2. 카메라 권한 **허용** 클릭
3. 양손을 카메라 앞에 보이게 위치

## 손 제스처 사용법

화면은 좌우 반전(거울 모드)으로 표시됩니다. 내 오른손이 화면 오른쪽에 보입니다.

### 오른손 — 멜로디 연주

| 동작 | 효과 |
|------|------|
| **손 높이 (위/아래)** | 음높이 결정. 손을 올릴수록 높은 음, 내릴수록 낮은 음 |
| **엄지-검지 핀치 (집기)** | 현재 음높이의 음을 연주 (누르는 동안 음 지속) |
| 핀치 풀기 | 음 종료 |

- 핀치 판정: 엄지 끝(4번)과 검지 끝(8번)의 거리 / 손바닥 크기 비율이 감도 슬라이더 값 미만이면 핀치로 인식
- 음높이는 현재 화음(CHORD)에 속한 음으로 자동 스냅되어 불협화를 방지

### 왼손 — 지휘 (없어도 연주 가능)

| 동작 | 효과 |
|------|------|
| **손 높이 (위/아래)** | 셈여림(강약) 제어. 높이 올릴수록 ff(매우 강하게), 내릴수록 p(여리게) |
| **손 좌우 위치** | 화음 진행 선택. 왼쪽=앞부분 화음, 오른쪽=뒷부분 화음 |
| **주먹 쥐기** | 페달 ON — 음이 울려 퍼짐, 리버브 증가 |
| **손 펼치기** | 페달 OFF — 페달 중에 울리던 음 일괄 종료 |
| 왼손 없음 | 셈여림 기본값(mf), 화음은 자동으로 진행 |

- 주먹 판정: 4개 손가락 끝(8·12·16·20번)의 평균 거리 / 손바닥 크기 비율 < 1.7

### 셈여림 표시 (화면 왼쪽 하단)

| 표시 | 범위 | 설명 |
|------|------|------|
| `ff` | 왼손 매우 높이 | 매우 강하게 |
| `f` | 왼손 높이 | 강하게 |
| `mf` | 왼손 중간 | 조금 강하게 (기본) |
| `mp` | 왼손 낮게 | 조금 여리게 |
| `p` | 왼손 매우 낮게 | 여리게 |

## 작곡가 스타일

왼쪽 패널에서 작곡가를 선택하면 BPM, 음계, 화음 진행, 반주 패턴이 바뀝니다.

| 작곡가 | 시대 | 조성 | 특징 |
|--------|------|------|------|
| **Bach** | 바로크 (BPM 84) | e단조 | 2~3성 대위, 8분음표 모터리듬, 5도 하행 시퀀스 |
| **Mozart** | 고전 (BPM 120) | C장조 | 알베르티 베이스 16분음표, 전반/후반 셈여림 대비 |
| **Chopin** | 낭만 (BPM 60) | E♭장조 | 녹턴식 아르페지오, 9화음, 루바토(랜덤 타이밍 미동) |
| **Scriabin** | 상징주의 (BPM 58) | 신비화음 | 온음음계 멜로디, 떨리는 트레몰로, 기능화성 없음 |
| **Rachmaninoff** | 후기낭만 (BPM 64) | c♯단조 | 광활한 아르페지오, 옥타브 종소리, 나폴리 6화음 |

## 컨트롤 패널

| 슬라이더 | 기능 |
|----------|------|
| **핀치 감도** (0.30~0.70) | 값이 클수록 핀치 인식이 쉬워짐. 손이 작거나 거리가 멀면 올림 |
| **전체 볼륨** | 마스터 게인 (0~100%) |
| **빠르기** | BPM 40~160. 작곡가 선택 시 자동 설정됨 |
| **● 녹음** | 클릭하면 녹음 시작, 다시 클릭하면 정지 후 .webm 파일로 다운로드 |

## 코드 구조 설명

### 전체 흐름

```
버튼 클릭
  → buildAudio()   : Tone.js 피아노 + 리버브 + 녹음기 초기화
  → startCamera()  : 웹캠 스트림 획득
  → buildHands()   : MediaPipe HandLandmarker 모델 로드
  → Tone.Transport.start() + render() 루프 시작
```

### 1. 손 인식 — MediaPipe HandLandmarker

`buildHands()`에서 MediaPipe Tasks Vision의 `HandLandmarker`를 GPU 모드로 초기화합니다. `render()` 루프에서 매 프레임 `detectForVideo(video, now)`를 호출해 손 랜드마크 21개(0=손목, 4=엄지끝, 8=검지끝, 12=중지끝 등)를 좌표로 받습니다.

**손 구분**: 랜드마크 배열을 손목 x좌표(화면 반전 후) 기준으로 정렬. 손이 하나면 오른손, 둘이면 왼쪽에 있는 것을 왼손으로 처리합니다.

```
CONN 배열: 랜드마크 번호 쌍 — 손가락 관절을 선으로 잇는 데 사용
mx(p) = W - p.x*W   // 좌우 반전 (거울 효과)
my(p) = p.y*H
```

### 2. 오른손 처리 — 멜로디

```javascript
const palm = dist(right[0], right[9])   // 손바닥 크기 (정규화 기준)
const ratio = dist(right[4], right[8]) / palm  // 엄지-검지 거리 비율
const isPinch = ratio < sens            // sens: 감도 슬라이더 값

const y = right[8].y                   // 검지 끝 y좌표 (0=상단, 1=하단)
const idx = Math.round((1-y) * (melody.length-1))  // 위=높은 인덱스
const target = snapToHarmony(idx, scale)  // 현재 화음에 스냅
```

`snapToHarmony()`는 선택된 음이 현재 화음(CHORD)에 없으면 ±1, ±2 인덱스 중 화음 구성음을 찾아 대체합니다. 이로써 손 위치가 정확하지 않아도 항상 어울리는 음이 납니다.

**디바운스**: `pendC` 카운터로 같은 음이 2프레임 이상 유지될 때만 실제 발음. 손 떨림으로 인한 오발음 방지.

### 3. 왼손 처리 — 지휘

```javascript
DYN = 0.3 + (1 - left[0].y) * 0.7     // 손목 높이 → 셈여림 (0.3~1.0)
leftXscreen = 1 - left[0].x            // 손목 좌우 → 화음 선택 인덱스

const open = (손가락4개 끝의 평균거리 / 손바닥크기)
PEDAL = open < 1.7                     // 주먹 = 손가락 접힘 = 페달 ON
```

페달이 켜지면 리버브 wet을 `+0.22` 올리고, 꺼질 때 페달 중 눌린 음(`sustained[]`)을 한꺼번에 해제합니다.

### 4. 오디오 엔진 — Tone.js

**신호 체인**: `piano(Sampler)` → `reverb(Reverb)` → `master(Gain)` → 스피커  
**동시에**: `master` → `recorder(Recorder)` (녹음용)

`Tone.Sampler`는 Salamander Grand Piano 샘플(실제 피아노 녹음)을 불러와 중간 음표를 피치 시프트로 보간합니다.

**루프**: `Tone.Loop`가 16분음표 단위로 반복 실행됩니다. 16스텝(= 1마디)마다 `bar`가 증가하고, 왼손이 있으면 좌우 위치로 화음을 선택, 없으면 순서대로 자동 진행합니다.

```javascript
const pos = loop._s % 16   // 마디 내 16분음표 위치 (0~15)
PHASE = bar / (bars-1)     // 곡의 진행도 (0.0~1.0), 셈여림 변화에 사용
S.step(pos, time)          // 작곡가별 반주 패턴 실행
```

### 5. 작곡가 스타일 객체 (STYLES)

각 작곡가는 다음 필드를 가집니다:

```javascript
{
  bpm: 템포,
  reverb: 리버브 기본 wet값,
  melody: [MIDI번호, ...],   // 오른손이 선택 가능한 음 목록
  prog: [[MIDI,...], ...],   // 8개 화음 진행 (각 화음 = 구성음 MIDI 배열)
  step(pos, time) { }        // 16분음표 위치마다 호출되는 반주 패턴
}
```

**step 함수 예시 (Bach)**:
- 짝수 박: 베이스 음 순환 (seq=[0,1,0,2])
- 홀수 박: 상성부 2개 음을 교대

**step 함수 예시 (Chopin)**:
- 짝수 박만 발음 (8분음표처럼 아르페지오)
- `seq=[0,1,2,3,4,5,4,3]`로 화음 위아래 굴리기
- `Math.random()*0.012`으로 타이밍 미세 흔들림 (루바토 효과)

### 6. 오선지 렌더링 — drawStaff()

화면 하단 80% 위치에 5선을 그립니다.

```
줄 위치:  top = H*0.80 - 2*gap   (gap = H*0.026)
1선(위): top
2선: top + gap   → D5 (MIDI 74)
3선(중): top + 2*gap → B4 (MIDI 71) ← 기준점
4선: top + 3*gap → G4 (MIDI 67)
5선(아래): top + 4*gap → E4 (MIDI 64)
```

**음표 Y 좌표 계산 (diatonic 기반)**:

반음 단위로 계산하면 E→F(반음)와 C→D(온음)가 같은 간격이 되어 오선과 안 맞습니다. 온음계(diatonic) 단계로 변환합니다:

```javascript
// 반음→온음계 단계 변환 테이블 (C=0, C#=0.5, D=1, D#=1.5, E=2, F=3, ...)
const dSteps = m => {
  const t = [0,.5,1,1.5,2,3,3.5,4,4.5,5,5.5,6];
  return Math.floor(m/12)*7 + t[m%12];
};
const yOf = m => mid - (dSteps(m) - dSteps(71)) * (gap*0.5);
//   B4 → mid (3선), D5 → top+gap (2선), F5 → top (1선) ✓
```

음표는 오른쪽(플레이헤드 x=W*0.80)에서 생성되어 매 프레임 1.15px씩 왼쪽으로 이동합니다.
